ISSUE 1 — wrong weightType used for transfer basket weighings
================================================================
File: src/app/transfers/[id]/TransferDetailManager.tsx (line ~109)

Current code:
  const fieldWeighTypeId = weightTypes.find((wt) => wt.name.includes("שטח"))?.id
    ?? weightTypes[0]?.id ?? "";

This matches "שקילת ניטור-שטח" (field monitoring) — wrong. Basket weighings created from
a transfer (דילול/פירוק/שיווק) should use the third seeded weight type, "שקילת אפיון טנק"
(tank characterization) — see prisma/seed.ts line ~444:
  const weightTypeNames = ["שקילת ניטור-שטח", "שקילת ניטור-רשת", "שקילת אפיון טנק"];

Fix:
- Change the `.includes("שטח")` match to `.includes("אפיון טנק")`.
- Rename the variable from `fieldWeighTypeId` to `tankWeighTypeId` (or similar) so this
  doesn't get misread again — it's currently named after the wrong concept.
- Update the prop name passed into <BasketWeighingModal weightTypeId={...} /> at the bottom
  of the same file accordingly (prop name itself can stay `weightTypeId`, just fix the
  variable feeding it).
- No schema change needed — `שקילת אפיון טנק` already exists in seed data.

Side effect to check: src/app/weighings/page.tsx filters the two tabs by
`weightType.name.contains("שטח")` and `contains("רשת")`. Tank-characterization weighings
(אפיון טנק) won't match either filter, which is correct — they stay visible only inside the
transfer detail screen where they're created, same as before. Don't add a third tab for
this unless asked; just confirm after the fix that the field/net tabs are no longer polluted
with transfer-basket weighings (currently they are, because of the bug above).
